# Phase 01 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none; manual Svelte + CSS in `src/styles/global.css` |
| Preset | not applicable |
| Component library | none |
| Icon library | inline SVG icons following existing `stock-watch-button` and footer icon patterns; use checklist/list icon only for this feature |
| Font | system stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |

Design system source: existing `:root` tokens in `src/styles/global.css`. Do not initialize shadcn for this phase; the app is Svelte, has no `components.json`, no Tailwind config, and no registry-backed component system.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge gaps, compact quantity-control dividers |
| sm | 8px | Presentation row gaps, item metadata gaps, panel inner groups |
| md | 16px | Default panel padding, drawer padding, list item padding |
| lg | 24px | Desktop panel gap from catalog, drawer section separation |
| xl | 32px | Desktop panel header-to-list breathing room when content is dense |
| 2xl | 48px | Minimum floating button block size and comfortable touch affordance |
| 3xl | 64px | Reserved bottom safe area for mobile floating button/drawer spacing |

Exceptions: icon-only and compact touch controls may be 44px minimum on mobile/coarse pointers; existing catalog image sizes remain 56px mobile and 64px desktop; desktop quote panel width is `minmax(320px, 360px)` when visible.

---

## Typography

Use exactly these four text sizes and two weights for new quote-list UI. Existing catalog typography may remain as-is.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px | 400 | 1.5 |
| Label | 13px | 700 | 1.35 |
| Heading | 17px | 700 | 1.2 |
| Display | 22px | 700 | 1.15 |

Typography contract: use sentence case for headings and controls. Do not use hero-scale text inside the panel or drawer; this is a compact planning tool embedded in the catalog.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f7` / `var(--bg)` | Page background and catalog area around the list |
| Secondary (30%) | `#ffffff` / `var(--surface)` plus `#eeeeef` / `var(--surface-soft)` | Quote side panel, drawer sheet, list item surfaces, quantity controls, notice backgrounds |
| Accent (10%) | `#006edb` / `var(--accent)` | Add `+1` action, active list count badge, focused quantity input, selected/active list affordances, mobile floating checklist button |
| Destructive | `#a15c00` / `var(--warn)` | Clear-list action, remove-on-decrement warning state, storage warning notices only |

Accent reserved for: presentation `+1` add button, active quantity-control focus ring, quote-list count badge, mobile floating checklist/list button, panel/drawer active highlights, non-blocking reconciliation notice accent border.

Do not use accent for every provider link, every metadata badge, or generic body text. Missing-data badges use muted/warn styling, not accent.

## Interaction Contract

| Area | Contract |
|------|----------|
| Add affordance | Each product presentation row shows a compact `+1` action. It belongs to the presentation, not to provider offers and not only to the base color/product card. |
| Add behavior | Tapping `+1` adds the presentation as `1 carrete`; if already present, increment by 1. Do not ask for provider selection in Phase 1. |
| Active list mode | After the first item exists, enable list mode: desktop panel appears; mobile floating checklist/list button appears with count; presentation rows may show richer quantity controls. |
| Desktop layout | When the list has items and viewport is desktop, render a right-side panel beside the catalog. Keep the catalog readable; panel width must be 320-360px and sticky within the viewport below the header/quick-line area. |
| Mobile layout | Use a fixed floating checklist/list button with item count. Tapping opens a bottom drawer with a visible drag/header area and close control. Do not use cart, bag, checkout, or full-screen modal metaphors. |
| Quantity controls | Use `- / editable integer / + / +6 / +12`. Minimum is 1. Decrementing below 1 removes the item. Editable input accepts whole integers only and clamps/normalizes invalid values. |
| Remove item | Provide an explicit remove icon/button per item in the panel/drawer. It removes immediately with no confirmation; clearing the whole list requires confirmation. |
| Clear list | A secondary/destructive action inside panel/drawer. Ask confirmation with the copy defined above. |
| Toggle controls | Include a persisted or session-stable toggle for extended controls. Hiding controls must not delete the list and must not stop autosave. |
| Autosave | Save every add, remove, quantity edit, clear, and relevant quote-list setting change automatically. |
| Storage failure | If `localStorage` throws or is unavailable, keep in-memory list for the current session and show the error copy inside the panel/drawer. |
| Reconciliation | On load after `stock.json`, remove saved items whose `product.id` no longer exists and show the non-blocking removal notice. |
| Out-of-stock presentations | Allow adding presentations with no online stock registered, but mark them with `confirmar stock` and keep the notice language non-commerce. |

## Component Contract

| Component / Surface | Required behavior |
|---------------------|-------------------|
| Presentation add control | Lives inside each `.presentation-row`; compact by default; uses `+1` visual label and descriptive `aria-label`. |
| Quote side panel | Desktop-only surface shown only when item count > 0; title, count, notices, items, clear action, and storage/reconciliation notices live here. |
| Mobile floating button | Fixed bottom action with checklist/list icon and numeric count; hidden when item count is 0; respects bottom safe area and does not cover content controls. |
| Mobile bottom drawer | Contains the same list content as desktop panel; close button must be reachable, Escape closes when supported, backdrop click may close. |
| Quote list item | Shows product name, brand, diameter, presentation, article/SKU/EAN when present, quantity, missing-data badges, and remove action. |
| Quantity stepper | One stable-width control group; numeric input uses tabular numbers where available and does not resize the row when values change. |
| Notices | Local-only and non-commerce notices appear below the panel/drawer title before list items. Reconciliation/storage notices appear below them and are non-blocking. |

## Responsive Contract

| Viewport | Contract |
|----------|----------|
| `> 960px` | Catalog uses a two-column application layout only when quote list has items: main catalog + right panel. Without items, preserve current full-width catalog. |
| `821px-960px` | Prefer drawer/floating button if the panel would make product cards cramped; do not force a narrow side panel. |
| `<= 820px` | Use mobile floating button and bottom drawer. Drawer max height is 82vh; body behind drawer does not scroll unexpectedly while drawer is open. |
| `<= 520px` | Quantity controls wrap cleanly into two rows if needed: `- input +` first, `+6 +12` second. No text may overflow a button. |

## Accessibility Contract

| Concern | Contract |
|---------|----------|
| Naming | Buttons use explicit accessible names: `Agregar 1 carrete...`, `Sumar 6 carretes...`, `Quitar...`, `Abrir lista de cotizacion`. |
| Icon metaphor | Checklist/list icon only. Do not use shopping cart, bag, checkout, credit-card, or order icons. |
| Keyboard | All actions are buttons or inputs; tab order follows title/notices/items/actions; Escape closes mobile drawer and confirmation where implemented. |
| Focus | Use visible focus ring based on `var(--accent)` with at least 2px outline or equivalent box-shadow. |
| Live updates | Item count changes and storage/reconciliation warnings use polite live regions. |
| Touch targets | Mobile/coarse-pointer controls use at least 44px target size, even when visual button is compact. |
| Confirmation | Clear-list confirmation must be reachable by keyboard and must not trap focus after cancellation. |

---

## Data Display Contract

| Field | Display rule |
|-------|--------------|
| Product name | Use base product name; avoid duplicating presentation text in the title if already shown separately. |
| Material/line | Show normalized line/material when available; otherwise show `confirmar dato`. |
| Color | Show color text and existing swatch behavior when available. |
| Brand | Show brand; if missing, show `sin marca` only inside metadata, not as a headline. |
| Diameter | Show existing `diameterLabel` output; if missing, badge `sin diametro`. |
| Presentation | Show product presentation as package descriptor, not as quote quantity. If unavailable, badge `sin presentacion`. |
| Article code/SKU/EAN | Show the most provider-facing identifier available. If none exists, badge `sin codigo`. |
| Quantity | Show as `{n} carrete` / `{n} carretes`; never show quote-list quantity as kg. |
| Stock status | Show `confirmar stock` for no online stock registered or unknown stock; do not promise availability. |
